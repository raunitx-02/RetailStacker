Executive Summary

This proposal outlines the development of the RetailStacker Chrome Extension — a browser-native intelligence tool that injects powerful seller analytics directly into Amazon India's product pages and search results. Inspired by the functionality of Helium 10's Chrome Extension, this tool is purpose-built and specifically calibrated for the Indian e-commerce market, covering Amazon India, Flipkart, and Meesho.

Since the core intelligence engine, API integrations (Keepa, FBA fee calculations, revenue estimation models), and user authentication infrastructure of RetailStacker are already fully operational, the development effort required is significantly reduced, allowing a highly polished, production-ready MVP to be delivered within an estimated 30 to 36 development hours.

2
The Problem We're Solving

Today, Indian Amazon sellers are forced to switch between their seller dashboard, RetailStacker's web platform, and Amazon itself to gather critical market intelligence. This fragmented workflow wastes time and causes sellers to miss real-time opportunities. Global competitors like Helium 10 offer a browser extension that eliminates this friction, but it is not optimised for Indian pricing, logistics, GST, or regional marketplaces.

⏱️
Time Lost

Sellers toggle between 4–6 tabs to manually compile product intelligence that should be instant.

🌐
Global Tools, Indian Gap

No existing extension is built for Amazon India's fee structure, regional shipping, and GST compliance.

📉
Missed Opportunities

Without real-time competitor data on the page, sellers list at the wrong price and lose the Buy Box.

3
Proposed Solution

The RetailStacker Chrome Extension will be a Manifest V3 (MV3) compliant browser extension that embeds a premium, dark-mode intelligence layer directly within Amazon India. The extension will communicate with the existing RetailStacker backend, ensuring all data is live, accurate, and gated behind the user's active subscription plan.

  • 🔍
    Xray — Live Product Intelligence: On any Amazon.in search page, a floating "Scan with RetailStacker" button overlays an interactive data table showing BSR, estimated monthly sales, revenue, active seller count, and review velocity for every product on the page.
  • 👥
    Active Sellers Counter: Instantly reveals whether a listing is Private Label (1 seller) or Wholesale / Arbitrage territory (multiple sellers). A critical signal for sourcing decisions that no Indian tool currently provides inline.
  • 💰
    India-Specific Profit Calculator: On individual product pages, a pop-up calculator uses the product's actual weight and dimensions to compute exact Amazon India referral fees, FBA/Easy Ship fulfillment costs, and net profit margin.
  • 📋
    Bulk ASIN Grabber: A one-click button that copies all ASINs from any search results page to the clipboard, ready to paste into RetailStacker's Cerebro or Bulk ASIN Analyzer tool.
  • 📦
    Inventory Level Indicator: Displays an estimated inventory depth badge on each listing card, helping sellers identify when a competitor is running low — a prime opportunity to capture the Buy Box.
  • 🔒
    Subscription-Gated Access: The extension is linked to the user's existing RetailStacker account. Only active Growth or Diamond plan subscribers can access full features, driving plan upgrades and reducing churn.

Development Timeline & Phases

Because the RetailStacker backend (Keepa integration, authentication, revenue models, FBA fee calculators) is already fully built and operational, the development scope is focused on the extension's UI layer, content injection scripts, subscription gating logic, and dedicated API endpoints. The estimated total build time is 30 to 36 development hours.

1
Phase 1 — Week 1
Foundation, Auth & Project Setup
⏱ 6 – 8 Hours
Initialize the Plasmo + React project with TypeScript. Configure Tailwind CSS with the RetailStacker dark-mode design system and brand tokens. Build the extension popup with a premium login/signup form connected to the existing RetailStacker user authentication API. Implement JWT token storage in Chrome's secure local storage. Set up the extension manifest, permissions, and icon assets for Chrome Web Store compliance.
2
Phase 2 — Week 1–2
Amazon DOM Scraping & API Bridge
⏱ 8 – 10 Hours
Write the Content Script that runs invisibly on amazon.in. It will intelligently detect search results pages vs. individual product pages, extract all ASINs from the DOM without page reloads, and pass them to a dedicated new API route (/api/extension/xray) on the RetailStacker backend. Handle rate limiting, error states, and CORS. The endpoint will return enriched live product data via Keepa. Thorough cross-page testing on amazon.in across multiple search categories.
3
Phase 3 — Week 2–3
Xray Overlay UI & Data Table
⏱ 8 – 10 Hours
Build the animated, floating Xray data table that injects seamlessly over Amazon India search results. Columns include: Product Name, Current Price, BSR, Est. Monthly Sales, Est. Monthly Revenue, Active Seller Count, Review Count, and Rating Score. Implement full column sorting, hover tooltips, subscription-tier feature gating with upgrade prompts, and one-click CSV export. Ensure the overlay is fully responsive and does not interfere with Amazon's native page layout or performance.
4
Phase 4 — Week 3–4
Profit Calculator, QA, Polish & Submission
⏱ 8 – 10 Hours
Inject the India-specific Profit Calculator sidebar on individual product pages, using real Amazon India referral fee slabs, Easy Ship / FBA fulfillment costs, and GST. Conduct full end-to-end QA across Chrome versions and OS platforms. Final UI polish — micro-animations, loading skeletons, and error states. Package the production build. Prepare Chrome Web Store listing assets (promotional images, screenshots, store description). Submit for Google review and handle any policy compliance feedback.
